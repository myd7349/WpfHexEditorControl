🔧 Main Toolbar Integration (Dynamic)

La Main Toolbar existe déjà.

Les fonctionnalités liées au BuildSystem doivent ajouter dynamiquement leurs commandes dans cette toolbar.

Les boutons sont enregistrés via le Command System de l’IDE.

Bouton	Action

BuildSystem (#103)

CanExecute = () => SolutionManager.HasOpenSolution

⚙ Configuration Manager Integration

Le Configuration Manager permet de gérer les configurations de compilation pour les projets et solutions.

Il est intégré à l’IDE et utilise les données du ProjectSystem et du BuildSystem (#103).

Objectifs :

choisir la configuration active

sélectionner la plateforme cible

définir le projet de démarrage

Configurations supportées

Chaque projet possède plusieurs configurations :

Debug
Release
Custom configurations

Structure interne :

Solution
 └─ Project
      └─ BuildConfiguration
           ├─ Name (Debug / Release)
           ├─ Platform (AnyCPU / x64 / x86)
           ├─ OutputPath
           └─ BuildFlags

Classe associée :

class BuildConfiguration
{
    string Name;
    string Platform;
    string OutputPath;
    bool Optimize;
}
UI Integration

Le Configuration Manager peut être accessible via :

Menu
Build
 └─ Configuration Manager
Toolbar Dropdown

Dans la Main Toolbar existante :

[Configuration ▼]  Debug / Release
[Platform ▼]       AnyCPU / x64

Ces contrôles doivent être dynamically populated à partir du ProjectSystem.

Flux d'utilisation
User selects Debug configuration
          ↓
ConfigurationManager
          ↓
ProjectSystem
          ↓
BuildConfiguration updated
          ↓
BuildSystem uses selected configuration
Interaction avec BuildSystem

Lors d’un build :

BuildSystem
      ↓
Active Configuration
      ↓
MSBuild Adapter
      ↓
Compiler

Exemple MSBuild :

Configuration=Debug
Platform=AnyCPU
📑 Document Tabs System

Le Document Tabs System gère tous les fichiers ouverts dans l’IDE.

Chaque fichier ouvert devient un Document Tab dans l’éditeur principal.

Ce système est déjà présent dans l’IDE et doit être connecté au :

DocumentManager

WHFMT Detection System (#81)

EditorResolver

Structure
Editor Tabs
 ├─ Program.cs
 ├─ Project.csproj
 ├─ firmware.bin
 ├─ script.wh
 └─ image.png

Chaque tab représente un Document.

Classe typique :

class Document
{
    string Path;
    string Name;
    IEditor EditorInstance;
    bool IsDirty;
}
Ouverture d’un document

Flux :

User opens file
      ↓
DocumentManager
      ↓
WHFMT Detection System (#81)
      ↓
EditorResolver
      ↓
Create editor instance
      ↓
Open document tab
Routing des éditeurs

Le WHFMT Detection System choisit l’éditeur approprié.

Exemples :

Type de fichier	Editor
.cs / .vb	CodeEditor
.json / .xml	CodeEditor
.bin	HexEditor
.exe / .dll	HexEditor + DisassemblyViewer
.png / .bmp	ImageViewer
.wav	AudioViewer
.tbl	TblEditor
structure overlays	StructureEditor
États des documents

Les tabs doivent gérer les états suivants :

Clean
Modified
ReadOnly
BinaryMode

Indication visuelle :

(* = document modifié)

Actions disponibles

Sur chaque tab :

Close
Close All

Pin Tab

       ↓

       ↓
Jump to Line

     ↓

🧩 UI Contribution System Integration

L’IDE possède déjà un UI Contribution System permettant aux plugins et modules d’injecter dynamiquement des éléments dans l’interface utilisateur.

Ce système permet d’ajouter :

menus

boutons de toolbar

panels dockables

context menus

commandes

Il est utilisé par les modules suivants :

Plugins (#138)
BinaryAnalysis
Build.MSBuild
SolutionLoader
Roslyn

Les nouveaux systèmes introduits dans ce plan doivent s’intégrer à ce mécanisme existant, et ne doivent pas modifier directement l’UI.

🧠 Architecture
Plugin / Module
        ↓
UI Contribution Definition
        ↓
Existing UI Contribution System
        ↓
IDE UI (Menus / Toolbar / Panels)

Les composants principaux restent :

MenuBar
MainToolbar
SolutionExplorer
EditorTabs
ErrorList
OutputPanel
StatusBar
