⚙ Configuration Manager Integration

Le Configuration Manager permet de gérer les configurations de compilation pour les projets et solutions.

Il est intégré à l’IDE et utilise les données du ProjectSystem et du BuildSystem (#103).

Objectifs :

choisir la configuration active

sélectionner la plateforme cible

définir le projet de démarrage

Configurations supportées

Chaque projet possède plusieurs configurations :

Debug
Release
Custom configurations

Structure interne :

Solution
 └─ Project
      └─ BuildConfiguration
           ├─ Name (Debug / Release)
           ├─ Platform (AnyCPU / x64 / x86)
           ├─ OutputPath
           └─ BuildFlags

Classe associée :

class BuildConfiguration
{
    string Name;
    string Platform;
    string OutputPath;
    bool Optimize;
}
UI Integration

Le Configuration Manager peut être accessible via :

Menu
Build
 └─ Configuration Manager
Toolbar Dropdown

Dans la Main Toolbar existante :

Ces contrôles doivent être dynamically populated à partir du ProjectSystem.

 ├─ Program.cs

 └─ image.png

🎛 Command System Integration (#78)
Rôle dans l’architecture

Le Command System agit comme un bus central d’exécution d’actions dans l’IDE.

Toutes les actions utilisateur passent par ce système :

menus

toolbar

raccourcis clavier

context menus

plugins

automatisation

Le Command System garantit que toutes les actions sont exécutées de manière uniforme.

🏗 Position dans l’architecture
IDE
│
├─ CommandSystem (#78)
│
├─ WorkspaceSystem
├─ SolutionSystem
├─ ProjectSystem
├─ BuildSystem
├─ Diagnostics
├─ EditorSystem
└─ Plugins (#138)

Le Command System sert d’interface commune entre :

UI
Plugins
Core Systems
🧠 Workflow général

Lorsqu’une action est déclenchée :

User Action
   ↓
UI Element (menu / toolbar / shortcut)
   ↓
Command System (#78)
   ↓
Command Handler
   ↓
Subsystem (BuildSystem / Editor / Plugin / etc.)

Exemple :

User clicks "Build Solution"
      ↓
CommandSystem.Execute("BuildSolution")
      ↓
BuildSystem.BuildSolution()
🔧 Utilisation dans l’UI

Tous les éléments UI existants utilisent le Command System :

UI Component	Command
Menu	Command
Toolbar	Command
Context Menu	Command
Shortcut	Command

Exemple :

Menu → Build → Build Solution
Toolbar → Build button
Shortcut → Ctrl+Shift+B

Ces trois éléments déclenchent la même commande :

BuildSolution
🧩 Structure d’une Commande

Une commande possède :

CommandId
Name
Execute()
CanExecute()

Exemple :

CommandDefinition
{
    Id = "BuildSolution",
    Execute = () => BuildSystem.BuildSolution(),
    CanExecute = () => Workspace.HasActiveSolution
}
