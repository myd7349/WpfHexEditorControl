# 1 AMÉLIORATION DU SOLUTION EXPLORER (BASES SÉCURISÉES)

## Objectif
Fournir un **Solution Explorer robuste et performant** tout en sécurisant les bases existantes :  

- Multi-projets par workspace (.whsln)  
- Fichiers multi-types, virtual folders et dossiers physiques (.whproj)  
- Configuration d’éditeur par fichier  
- Compatibilité avec CodeEditor, TextEditor, Folding et plugins  
- Support pour fichiers volumineux et workspaces complexes  

---

## 1️⃣ Structure et navigation

- Affichage hiérarchique : Workspace → Project → Virtual Folders → Items → Dossiers physiques  
- Virtual folders servent uniquement à **classer des projets ou items logiques**, pas à déplacer des fichiers physiques  
- Déplacement de fichiers entre dossiers physiques avec update automatique du fichier disque  
- Visualisation claire des types de fichiers (icônes + overlays) : Binary, Tbl, Image, FormatDefinition, .whlang, etc.  
- Indicateur de fichier “default item” (`defaultTblItemId`)  
- Tri et filtrage : type, extension, date modifiée, taille  
- Favoris / Quick Access pour fichiers et dossiers fréquents  
- Multi-select avec drag & drop et opérations groupées  
- **Tracking du document actif** (workspace-aware) configurable via OptionsPanel  

---

## 2️⃣ Gestion des fichiers et projets

- Création, renommage, déplacement, duplication de fichiers et virtual folders  
- Détection automatique des modifications externes (watcher de fichiers)  
- Priorisation des fichiers : user-created > imported > build-in  
- Sauvegarde automatique des métadonnées et de la configuration éditeur (`editorConfig`)  
- Support multi-tab pour ouvrir plusieurs fichiers simultanément  
- **Drag & Drop** depuis l’explorateur Windows et CTRL+C/X/V  

---

## 2️⃣1 Gestion améliorée des fichiers `.whchg` (WpfHexEditor Change)

- Chaque fichier binaire ouvert dans un workspace possède un fichier associé `.whchg`  
  - Activation par défaut, désactivable via OptionsPanel  
- Sérialisation **en temps réel** des modifications dans l’HexEditor sans toucher au fichier disque  
- Comportement hors workspace : CTRL+S applique directement sur le fichier disque  
- Caching intelligent et flush asynchrone pour performance  
- Undo/redo appliqué sur les modifications `.whchg`  
- Priorité : contenu du `.whchg` > fichier binaire réel  
- EventBus reçoit toutes les modifications pour audit et hooks plugin  
- Compatible multi-tab, multi-document, split interne  

---

## 2️⃣2 Gestion des fichiers `.whlang` avec tag `default`

- **Unique par type de fichier** : 1 fichier `.whlang` marqué `default` par format, enregistré dans le `.whproj`  
- Plusieurs fichiers `.whlang` peuvent exister, mais **un seul default**  
- CodeEditor / TextEditor :  
  - Vérifie d’abord le fichier `default`  
  - Si absent, prend le premier importé  
  - Sinon fallback sur build-in  
- OptionsPanel :  
  - Liste tous les `.whlang` importés par type de format  
  - Permet de changer le tag `default`  
  - Permet d’activer/désactiver l’override build-in  
- Cache pré-parsed pour accélérer l’éditeur  

---

## 3️⃣ Intégration avec CodeEditor, TextEditor et plugins

- Chaque fichier ouvert est workspace-aware et prêt pour folding, diagnostics, snippets  
- Clic droit sur fichier/folder :  
  - Open with CodeEditor / TextEditor  
  - GoToDefinition  
  - FindReferences  
  - RenameSymbol  
  - Run Script / Command via CommandSystem  
  - Save physical file (CTRL+S) configurable par type de fichier ou global  
- Plugins peuvent ajouter actions contextuelles, diagnostics ou export personnalisés  
- EventBus pour audit et undo/redo global  

---

## 4️⃣ UI / UX

- Dockable, flottant et redimensionnable, mémoire de position et état des panels  
- Quick Preview inline pour fichiers supportés  
- Collapsing automatique des virtual folders avec mémoire de l’état  
- Inline renaming avec validation  
- Notifications contextuelles pour erreurs, conflits ou diagnostics  
- OptionsPanel complet :  
  - Thèmes (clair/sombre)  
  - Couleurs syntaxe, fond, highlights  
  - Indentation, tab size, polices  
  - Tracking document actif par workspace  
  - Activation/désactivation `.whchg`  
  - Configuration par type de fichier (save mode, override build-in, default `.whlang`)  

---

## 5️⃣ Performance et sécurité

- Virtualization des nodes pour éviter le lag sur centaines de fichiers  
- Lazy loading pour projets/folders non ouverts  
- Caching des icônes, métadonnées et `.whchg` pour accélérer l’affichage  
- Multi-thread pour parsing JSON, folding, IntelliSense preload, flush `.whchg`  
- Validation JSON sécurisée à l’ouverture de `.whsln` / `.whproj`  
- Vérification des IDs uniques (`id`) pour éviter collisions ou doublons  

---

## 6️⃣ Modules principaux

- SolutionExplorerManager : gestion globale, drag-drop, tracking document actif  
- WorkspaceAdapter : multi-workspace, `.whchg`, `.whlang default`  
- FileTreeView : UI + virtualization + gestion dossiers physiques et virtual folders  
- LanguageIntegrationAdapter : `.whlang`, folding, diagnostics, snippets  
- EditorConfigAdapter : configuration editorConfig par fichier  
- CommandAdapter / EventAdapter : audit et undo/redo global  
- PluginIntegrationAdapter  
- SearchFilterEngine  
- OptionsPanelController : toutes les options pour éditeur, `.whchg`, tracking, thèmes, couleurs, tabs, save-mode  

---

## Résultat attendu

- Solution Explorer **robuste, performant et sécurisé**  
- Multi-projets et multi-workspace opérationnels  
- Gestion complète des virtual folders, dossiers physiques et items multi-types  
- Gestion `.whchg` performante pour gros fichiers  
- Gestion `.whlang` avec **tag `default` unique par type**, import automatique et fallback build-in  
- Base solide pour intégrer CodeEditor, TextEditor, IntelliSense, debugging et plugins  
- OptionsPanel complet et flexible pour toutes les fonctionnalités et comportements d’éditeur

# 2 CODEEDITOR VS-LIKE & TEXTEDITOR LIGHT (GLYPHRUN & PERFORMANCE)

## Objectif
Créer deux éditeurs robustes et extensibles, fonctionnels **workspace** et **standalone**, avec :  

- `CodeEditor` : full-featured VS-like  
  - Support complet `.whlang` avec override build-in  
  - Folding, snippets, diagnostics, hooks vers Solution Explorer  
  - Personnalisation via OptionsPanel (thèmes, couleurs, indentation, tab size, police)  
  - **Split interne sur le même fichier/document** via bouton scrollbar  
  - **Zoom intégré** (hérité JsonEditor)  
  - **Rendu texte optimisé** : exploration de **GlyphRun** pour maximiser les performances par rapport à FormattedText  
  - Support complet des `.whchg`  
- `TextEditor` : light version  
  - Mode texte simplifié pour fichiers standards (.txt, .log, .cfg…)  
  - Fonctionnalités avancées désactivées ou minimal  
  - Split interne et zoom maintenus  
  - Support `.whchg` si applicable  

---

## 1️⃣ Architecture / Namespaces

- Base commune : `WpfHexEditor.Editor.Core`  
  - Undo/Redo, Caret, Selection, Scroll, File I/O, Events  
  - Docking integration (multi-tab / split-view)  
  - Multi-workspace support, zoom intégré, base pour GlyphRun  
  - Support `.whchg` pour tous les types d’éditeur  

- Full Editor : `WpfHexEditor.Editor.CodeEditor`  
  - Hérite de Core  
  - Support complet `.whlang` avec priorité `default`  
  - Folding, snippets, diagnostics complets  
  - Intégration forte avec Solution Explorer (workspace aware)  
  - Split interne par document  
  - OptionsPanel pour personnalisation  
  - Rendu texte optimisé avec exploration GlyphRun  

- Light Editor : `WpfHexEditor.Editor.TextEditor`  
  - Hérite de CodeEditor  
  - Fonctionnalités avancées désactivées  
  - Split interne et zoom conservés  

---

## 2️⃣ Support multi-langages et fichiers

- `CodeEditor` :  
  - Langages build-in et user-imported (.whlang) avec tag `default`  
  - Fichiers texte standard et structurés (.txt, .log, .json, .tbl, .whlang…)  
  - Folding et snippets basés sur `.whlang`  
  - EditorConfig par fichier : bytesPerLine, encoding, editMode, scrollOffset, caret, selection, firstVisibleLine  
  - Support complet `.whchg`  

- `TextEditor` :  
  - Mode texte light, syntaxe highlight minimale si disponible  
  - Folding et snippets désactivés  
  - Support `.whchg` pour documents binaires si applicable  

---

## 3️⃣ UI / UX

- Docking déjà géré par `WpfHexEditor.Docking`  
- Split interne par document via bouton haut de scrollbar  
- Quick preview pour fichiers supportés  
- Visualisation de lignes, indentation, whitespace  
- Multi-selection et drag/drop  
- Inline renaming et search/replace simple  
- Zoom intégré (hérité JsonEditor)  
- Personnalisation via OptionsPanel :  
  - Thèmes (clair / sombre)  
  - Couleurs syntaxe, fond, highlights  
  - Indentation, taille tab, polices  
  - Gestion `.whchg` et `.whlang default`  
  - Tracking document actif par workspace  

---

## 4️⃣ Modes d’usage

- **Workspace aware** : navigation projet / workspace, folding/snippets selon `.whproj`, diagnostics contextuels, `.whchg`  
- **Standalone** : ouverture fichier unique, folding/snippets activés si structuré, split interne et zoom disponibles, pas d’interaction Solution Explorer, `.whchg` désactivé ou save direct  

---

## 5️⃣ Performance

- Virtualization des lignes pour gros fichiers (>1GB)  
- Parsing multi-thread pour folding et syntaxe  
- **Optimisation rendu texte avec GlyphRun** pour remplacer ou compléter FormattedText  
- Caching tokens et folding regions  
- Optimisation mémoire et rapidité pour TextEditor  
- Gestion `.whchg` performante sans blocage UI  

---

## 6️⃣ Modules principaux

- CodeEditorManager  
- LanguageIntegrationAdapter (.whlang, folding et diagnostics)  
- EditorConfigAdapter  
- SnippetsManager (CodeEditor complet, désactivé dans TextEditor)  
- DiagnosticsManager (CodeEditor complet, minimal ou désactivé dans TextEditor)  
- FileViewer (text, binary, image preview simple)  
- ThemeManager  
- EventAdapter / CommandAdapter  
- SplitViewController  
- ZoomController  
- GlyphRunRenderer / TextRenderAdapter (expérimentation performance)  

---

## Résultat attendu

- `CodeEditor` : full-featured, multi-langages, folding, snippets, diagnostics, intégré ou standalone  
- Support complet `.whlang` avec folding et snippets spécifiques, tag `default` respecté  
- Split interne sur le même fichier activable via bouton scrollbar  
- Zoom fonctionnel et intégré  
- **Rendu texte optimisé via GlyphRun pour maximum performance**  
- Personnalisation complète via OptionsPanel, y compris `.whchg`, tracking et overrides  
- `TextEditor` : light, rapide, stable, support fichiers texte standard, split interne et zoom maintenus, support `.whchg` si applicable
