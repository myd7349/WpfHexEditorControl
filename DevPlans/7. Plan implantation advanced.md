1️⃣ Solution System (#101, #102, #107, #138)

Interfaces existantes :

ISolution
ISolutionManager
IProject
IProjectItem

Actions détaillées :

- Étendre ces interfaces pour gérer code source, ressources et compilation
- Les formats de solutions sont chargés via des Solution Loaders extensibles
- WHFMT Detection System (Core) :
     Détecte type et signature (.sln, .csproj, .vbproj, .whsln, .whproj)
     Fournit preferredEditor (code-editor, hex-editor)
     Valide offset, min_file_size, signature
     Fournit métadonnées : format, version, category, références

Loaders dédiés :

- WpfHexEditor.Plugins.SolutionLoader.VS → .sln, .csproj, .vbproj
- WpfHexEditor.Plugins.SolutionLoader.WH → .whsln, .whproj

Structure en mémoire :

Solution
│
├─ Project
│ ├─ SourceFiles
│ ├─ Resources
│ ├─ References
│ └─ BuildOutput
└─ GlobalSettings

Événements associés :

SolutionOpened (#80)
ProjectOpened (#80)
Publication sur EventBus pour UI et plugins

3️⃣ Build System (#103, #138)

Module : WpfHexEditor.BuildSystem

Responsabilités :

     - Orchestration des builds multi-projets
     - Pipelines de build modulaires
     - Collecte des diagnostics (#96)
     - Logging des outputs (#40, #99) vers OutputPanel
     - Publication des erreurs/warnings vers ErrorList (#40, #96)

Pipeline détaillé :

Build Solution
     ↓
BuildSystem
     ↓
MSBuild API
     ↓
Compiler
     ↓
DiagnosticsProvider
     ↓
OutputPanel (logs MSBuild)
     ↓
ErrorList (erreurs/warnings)

Interface clé : IBuildSystem
Méthodes : BuildSolution(), BuildProject(), CleanProject(), RebuildProject()
Events : BuildStarted, BuildSucceeded, BuildFailed (#80)

4️⃣ MSBuild Integration (#103)

Plugin : WpfHexEditor.Plugins.Build.MSBuild
     - BuildSystem reste dans le Core
     - Plugin fournit l’adaptateur MSBuild

Responsabilités :

     - Charger et parser .csproj / .vbproj
     - Exécuter MSBuild via Microsoft.Build.Execution
     - Capturer outputs → OutputPanel (#40, #99)
     - Capturer diagnostics → ErrorList (#40, #96)

Classes et API :

     - ProjectCollection (#102)
     - BuildManager (#103)
     - ILogger / FileLogger (#40)

6️⃣ ErrorList Integration (#40, #96)

     - Reçoit tous les messages de compilation, warnings et messages d’analyse

7️⃣ Output Panel (#40, #99) inclus actuelement, valider les améliorations nécessaire

8️⃣ Code Editing (#84, #85, #86, #88, #89)

     - CodeEditor devient éditeur principal
     - Support : C#, VB.NET, JSON, XML, scripts...  via les fichier de détection .whlang actuellement supporté
     - Features : syntax highlighting, auto-indentation, symbol navigation, diagnostics intégrés (#84, #85, #86)
     - Support snippets dynamiques et multi-caret (#88)
     - AI-assisted code suggestions (#89) (si possible sans dépendance externe sinon met sur la glace)

WHFMT Detection System garantit l’ouverture du bon éditeur (#81)

9️⃣ Roslyn Integration (optionnel, #89, #94)

     - IntelliSense, code completion, refactoring (#89, #94)
     - Syntax analysis pour CodeEditor (#85, #86)
     - Support C#, VB.NET et scripts WH (#84, #85)
     - Mapping diagnostics vers ErrorList / OutputPanel (#96, #40)
