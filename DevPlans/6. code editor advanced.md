# 138 IN-IDE PLUGIN DEVELOPMENT — BUILD, TEST & SHIP PLUGINS WITHOUT LEAVING THE IDE

## Objectif

Permettre aux développeurs de **créer, compiler, déboguer et tester leurs propres plugins SDK** directement dans l’IDE WpfHexEditor, sans outillage externe :

- Utilisation du **Project System** (`.whsln` / `.whproj`) pour gérer les projets de plugins
- Templates de scaffolding dédiés aux plugins SDK
- Compilation via **MSBuild API (#103)** intégrée
- Chargement live / hot-reload du plugin en développement dans l’IDE courant
- Accès au **SDK IntelliSense** complet : `IWpfHexEditorPlugin`, `IIDEHostContext`, `IUIRegistry`, services
- Intégration avec l’Output Panel, l’Error Panel et le DiagnosticsManager
- Workflow "write → build → test → iterate" 100% in-IDE

---

## Architecture générale

```
PluginDevWorkflow
├── PluginProjectTemplate        — scaffolding .whproj pour nouveau plugin SDK
├── PluginBuildOrchestrator      — invoque MSBuild API (#103) sur le projet plugin
├── PluginDevLoader              — AssemblyLoadContext collectible pour hot-reload
├── PluginDevSandbox             — isolation légère du plugin en test (crashs sans tuer l’IDE)
├── PluginDevDiagnostics         — capture console / exceptions / perf du plugin chargé
├── LiveTestHarness              — surface le plugin dans l’IDE comme s’il était installé
└── PluginDevOptionsPage         — options : auto-rebuild on save, log level, sandbox mode
```

---

## 1️⃣ Project Templates — Plugin Scaffolding

- Nouveau template "SDK Plugin" disponible dans le dialogue "New Project" (#87)
- Template génère :
  - `.whproj` ciblant `net8.0-windows`
  - Référence au `WpfHexEditor.SDK`
  - Classe starter `MyPlugin.cs` implémentant `IWpfHexEditorPlugin`
  - `plugin.manifest.json` pré-rempli (Id, Name, Version, Author, LoadPriority)
  - Fichier `README.md` avec guide de démarrage
- Options du template :
  - Type de plugin (Panel, Editor, Binary Analysis, Toolbar, Custom)
  - Activer sandbox mode par défaut
  - Inclure stubs pour IPluginWithOptions, IPluginEventBus

---

## 2️⃣ Build Integration — MSBuild In-IDE

- Le projet plugin utilise **MSBuild API (#103)** pour compilation directe depuis l’IDE
- Bouton "Build Plugin" dans la toolbar du Solution Explorer

- Support pour post-build events (copie automatique vers `bin\Plugins\`)

---

## 3️⃣ Live Loading — Hot-Reload & Test Harness

- Après un build réussi :

  - Enregistré dans le `PluginHost` comme plugin "Dev" (badge distinctif dans Plugin Manager)

- "Rebuild & Reload" : unload propre de l’ancienne version → rechargement de la nouvelle
- Aucun redémarrage de l’IDE requis
- Le plugin dev est marqué visuellement dans le Plugin Monitor avec label `[DEV]`

---

- Isolation légère via `PluginDevSandbox` :

  - Timeout sur `Init` / `Activate` / `Deactivate` configurable
  - Logs de toutes les exceptions levées par le plugin
- Option : basculer en "full sandbox" via NamedPipe (`PluginSandbox` #81) pour tests extrêmes
- Indicateur visuel dans la status bar : `Plugin Dev Mode: Active`
